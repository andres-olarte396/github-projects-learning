# Autoevaluación Fase 4: Gestión de Ramas, Pull Requests y Revisión de Código

## Checklist de temas aprendidos y competencias adquiridas

- [ ] Comprendo el flujo de trabajo con ramas (branching) y PRs.
- [ ] Sé crear ramas feature y mantenerlas sincronizadas.
- [ ] Aplico convenciones de mensajes de commit (Conventional Commits).
- [ ] Realizo PRs enlazados a issues y uso palabras clave de cierre.

- [ ] Aplico estrategias de merge (squash, rebase, merge commit).
- [ ] Integro pruebas mínimas en cada PR.
- [ ] Uso hooks pre-commit para validar código antes de subirlo.
- [ ] Participo en revisiones de código efectivas y constructivas.
